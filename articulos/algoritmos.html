¿Qué es un Algoritmo?

Un algoritmo es una secuencia finita y ordenada de pasos que permiten resolver un problema o realizar una tarea determinada.

Antes de escribir cualquier programa, es necesario pensar primero en el algoritmo, es decir, en la lógica que va a resolver el problema, sin importar todavía en qué lenguaje de programación se va a implementar.

Según Joyanes Aguilar (2020), el algoritmo es independiente tanto del lenguaje de programación en que se exprese como de la computadora que lo ejecute.

Características de un Algoritmo

  • Finito: debe terminar después de un número determinado de pasos.
  • Definido: cada paso debe estar claramente especificado, sin ambigüedades.
  • Ordenado: los pasos deben seguir una secuencia lógica.
  • Preciso: debe indicar exactamente el orden de realización de cada paso.
  • General: debe poder aplicarse a un conjunto de datos de entrada, no solo a un caso particular.

Entrada, Proceso y Salida

Todo algoritmo puede describirse mediante tres elementos básicos:

Entrada

Los datos que el algoritmo necesita para funcionar. Por ejemplo, una lista de números.

Proceso

El conjunto de pasos u operaciones que se aplican sobre los datos de entrada.

Salida

El resultado que se obtiene después de aplicar el proceso a los datos de entrada.

Pseudocódigo

El pseudocódigo es una forma de describir un algoritmo utilizando un lenguaje similar al humano, combinado con estructuras propias de la programación, sin seguir la sintaxis estricta de ningún lenguaje en particular. 

Inicio
    Leer numero1, numero2
    suma = numero1 + numero2
    Escribir suma
Fin

Su principal ventaja es que permite diseñar la lógica de un programa sin preocuparse todavía por los detalles de sintaxis de un lenguaje específico.

Diagramas de Flujo

Un diagrama de flujo es una representación gráfica de un algoritmo, en la que se utilizan símbolos estandarizados para indicar el inicio, el fin, los procesos, las decisiones y el flujo de ejecución.

Símbolo Significado
Óvalo Inicio o fin del algoritmo
Rectángulo Proceso u operación
Rombo Decisión o condición
Paralelogramo Entrada o salida de datos
Flecha Dirección del flujo

Complejidad Algorítmica

La complejidad de un algoritmo mide de forma aproximada cuántos recursos (principalmente tiempo) necesita para resolver un problema a medida que crece el tamaño de los datos de entrada.

Notación Big O

Es la forma más común de expresar la complejidad de un algoritmo. Indica cómo crece el tiempo de ejecución en el peor de los casos.

  • O(1): tiempo constante, no depende del tamaño de la entrada.
  • O(n): tiempo lineal, crece proporcionalmente al tamaño de la entrada.
  • O(n²): tiempo cuadrático, común en algoritmos con bucles anidados.
  • O(log n): tiempo logarítmico, típico de algoritmos de búsqueda eficientes.

Comprender la complejidad ayuda a elegir el algoritmo más adecuado, especialmente cuando se trabaja con grandes cantidades de datos.

Ejemplos en Python

Calcular el promedio de una lista 

numeros = [10, 20, 30, 40]

suma = sum(numeros)
promedio = suma / len(numeros)

print("Promedio:", promedio)

Buscar un número en una lista 

numeros = [4, 8, 15, 16, 23, 42]
buscado = 15

if buscado in numeros:
    print("El número fue encontrado")
else:
    print("El número no está en la lista")

Ordenar una lista 

numeros = [5, 2, 9, 1, 7]

numeros_ordenados = sorted(numeros)

print("Lista original:", numeros)
print("Lista ordenada:", numeros_ordenados)

Encontrar el mayor elemento 

numeros = [3, 8, 2, 19, 6]

mayor = max(numeros)

print("El mayor número es:", mayor)

Cada uno de estos ejemplos representa un algoritmo simple: primero se piensa en los pasos (entrada, proceso, salida) y luego se traduce esa lógica al lenguaje Python.

Dato Interesante

La palabra "algoritmo" proviene del nombre del matemático persa Al-Juarismi, quien en el siglo IX escribió tratados sobre métodos sistemáticos para resolver ecuaciones, sentando las bases de lo que hoy conocemos como algoritmos.
