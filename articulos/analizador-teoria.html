¿Qué es el Análisis Estático de un Archivo?

El análisis estático consiste en examinar un archivo sin ejecutarlo. En lugar de correr el programa para ver qué hace, se estudia su contenido "desde afuera": su tamaño, su tipo, el texto que contiene y ciertos patrones que puedan indicar un comportamiento sospechoso.

Este enfoque es el primer paso natural cuando se analiza un archivo desconocido, ya que es seguro: al no ejecutarlo, se evita el riesgo de infectar el equipo o activar código malicioso.

¿Qué se suele revisar en un análisis estático?

  • Información básica: nombre, extensión, tamaño, fechas.
  • Una huella digital (hash) que identifica de forma única al archivo.
  • El texto legible que contiene el archivo por dentro.
  • Palabras clave, URLs o direcciones IP relacionadas con actividad maliciosa.

Hashes y la Huella Digital de un Archivo

Un hash es el resultado de aplicar una función matemática a un archivo, que produce una cadena de caracteres de longitud fija. Esta cadena funciona como una "huella digital": dos archivos idénticos siempre producen el mismo hash, y con cambiar un solo byte del archivo, el hash resultante cambia por completo.

¿Para qué sirve en ciberseguridad?

  • Verificar que un archivo no ha sido modificado.
  • Identificar un archivo de forma única, sin necesidad de leer todo su contenido cada vez.
  • Comparar un archivo contra bases de datos conocidas de amenazas.

SHA-256

SHA-256 es uno de los algoritmos de hash más utilizados actualmente. Siempre genera una cadena de 64 caracteres en hexadecimal, sin importar si el archivo pesa unos cuantos bytes o varios gigabytes. 

import hashlib

texto = "archivo de prueba"
hash_resultado = hashlib.sha256(texto.encode())

print(hash_resultado.hexdigest())

Para archivos grandes, en vez de leer todo el contenido de una sola vez, es común leerlos por bloques (por ejemplo, de 4096 bytes) e ir actualizando el hash progresivamente, para no consumir demasiada memoria.

Extracción de Cadenas de Texto (Strings)

Muchos archivos, incluso los binarios (ejecutables, por ejemplo), contienen fragmentos de texto legible en su interior: mensajes de error, rutas de archivos, nombres de funciones, URLs, etc. Buscar y extraer ese texto es una técnica muy común en análisis estático, inspirada en la herramienta strings de Linux.

La idea general es recorrer el archivo byte por byte y quedarse únicamente con las secuencias de caracteres imprimibles que tengan una longitud mínima (por ejemplo, 4 caracteres), descartando el resto. 

texto_ejemplo = "Hola\x00\x01Mundo123"

caracteres_legibles = "".join(
    caracter for caracter in texto_ejemplo if caracter.isprintable()
)

print(caracteres_legibles)

Expresiones Regulares (Regex)

Una expresión regular es un patrón que describe cómo debe verse un texto, y permite buscar coincidencias dentro de una cadena mucho más larga. En Python se utilizan a través del módulo re.

Ejemplo: encontrar todas las palabras que empiezan con mayúscula 

import re

texto = "Python es un Lenguaje muy Usado en Ciberseguridad"

patron = re.compile(r"\b[A-Z][a-z]*\b")
coincidencias = patron.findall(texto)

print(coincidencias)

Este mismo principio permite construir patrones para detectar, por ejemplo, direcciones URL (secuencias que comienzan con http:// o https://) o direcciones IP (cuatro grupos de números separados por puntos), simplemente cambiando el patrón utilizado.

Indicadores de Compromiso (IOC)

Un Indicador de Compromiso, o IOC (por sus siglas en inglés, Indicator of Compromise), es cualquier dato que sugiere que un sistema o un archivo podría estar comprometido o ser malicioso.

Ejemplos comunes de IOC

  • Hashes de archivos maliciosos conocidos.
  • Direcciones IP asociadas a servidores de atacantes.
  • URLs utilizadas para descargar cargas maliciosas.
  • Nombres de comandos o herramientas del sistema usados de forma sospechosa (por ejemplo, intérpretes de comandos o utilidades de administración).

En un proyecto educativo, se puede construir una lista propia de palabras clave consideradas sospechosas, y luego revisar si alguna de ellas aparece dentro del texto extraído de un archivo. Cuantos más indicadores se encuentren, mayor es la sospecha de que el archivo podría ser malicioso.

Manejo de Archivos en Python

Para poder analizar un archivo primero es necesario leerlo desde el disco. Python permite abrir archivos en distintos modos: como texto o como datos binarios.

Lectura en modo texto 

with open("ejemplo.txt", "r") as archivo:
    contenido = archivo.read()

print(contenido)

Lectura en modo binario

Cuando no se sabe si un archivo contiene texto plano (como ocurre con muchos archivos ejecutables o comprimidos), se abre en modo binario, representando su contenido como una secuencia de bytes. 

with open("ejemplo.bin", "rb") as archivo:
    datos = archivo.read()

print(len(datos), "bytes leídos")

Información básica de un archivo

El módulo os permite obtener datos como el tamaño de un archivo o la fecha de su última modificación, útiles para construir un reporte. 

import os

info = os.stat("ejemplo.txt")

print("Tamaño en bytes:", info.st_size)

Generación de Reportes en HTML desde Python

Un programa en Python puede construir directamente el contenido de una página HTML, combinando texto fijo (la estructura de la página) con datos calculados durante la ejecución (como un resultado o un hash), mediante f-strings. 

nombre_archivo = "prueba.txt"
riesgo = "BAJO"

html = f"""
<html>
  <body>
    <h1>Reporte de {nombre_archivo}</h1>
    <p>Nivel de riesgo: {riesgo}</p>
  </body>
</html>
"""

with open("reporte.html", "w") as archivo:
    archivo.write(html)

De esta manera, un script puede generar automáticamente una página web con los resultados de su análisis, sin necesidad de crear el archivo HTML manualmente.

Niveles de Riesgo

Una forma sencilla de clasificar un resultado es contar cuántos indicadores sospechosos se encontraron y, según ese número, asignar una categoría.

Cantidad de Indicios Nivel de Riesgo
0 Bajo
1 a 2 Medio
3 o más Alto

Es importante recalcar que este tipo de clasificación es una simplificación educativa. Un análisis profesional real combina muchas más señales, y herramientas especializadas para llegar a una conclusión confiable.

Sobre el Proyecto

Uniendo todos los conceptos anteriores es posible construir un analizador básico de archivos sospechosos: un script que recibe un archivo, calcula su hash SHA-256, extrae el texto legible que contiene, busca palabras clave, URLs o direcciones IP sospechosas dentro de ese texto, y con base en la cantidad de hallazgos asigna un nivel de riesgo. Al final, genera tanto un resumen en la terminal como un reporte visual en HTML.

Qué no es este tipo de proyecto

Es fundamental ser claro sobre sus límites: esta clase de herramienta no es un antivirus ni reemplaza un análisis profesional. No ejecuta el archivo analizado (por eso es segura de usar), solo lee su contenido y busca coincidencias de texto. Es un proyecto educativo, pensado para practicar y aprender, no para tomar decisiones de seguridad reales.

Posibles mejoras futuras

  • Analizar todos los archivos de una carpeta de forma automática.
  • Incorporar un modelo básico de aprendizaje automático en lugar de solo contar palabras clave.
  • Comparar el hash obtenido contra bases de datos públicas de amenazas conocidas.

Dato Interesante

El algoritmo SHA-256 forma parte de la familia SHA-2, diseñada por la Agencia de Seguridad Nacional de Estados Unidos (NSA) y publicada en 2001. Hoy en día es uno de los algoritmos de hash más utilizados en el mundo, incluyendo su papel fundamental en la tecnología detrás de Bitcoin.
