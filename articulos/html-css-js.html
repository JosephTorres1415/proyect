¿Qué es el Desarrollo Web?

El desarrollo web es el proceso de crear sitios y aplicaciones que funcionan a través de un navegador. Se suele dividir en dos grandes partes:

  • Frontend: todo lo que el usuario ve e interactúa directamente (HTML, CSS, JavaScript).
  • Backend: la lógica del servidor, bases de datos y procesamiento que ocurre "detrás de escena".

Este artículo se enfoca en el frontend, y en particular en los dos pilares que definen el contenido y la apariencia de una página: HTML y CSS.

HTML: Estructura de una Página

HTML (HyperText Markup Language) es el lenguaje que define la estructura y el contenido de una página web. No es un lenguaje de programación, sino un lenguaje de marcado: se basa en etiquetas que envuelven al contenido para indicarle al navegador qué es cada cosa (un título, un párrafo, una imagen, una lista).

Estructura básica de un documento HTML 

<!DOCTYPE html>
<html lang="es">

<head>
    <meta charset="UTF-8">
    <title>Mi Página</title>
</head>

<body>
    <h1>Hola Mundo</h1>
    <p>Este es mi primer párrafo.</p>
</body>

</html>

Explicación de cada parte

  • <!DOCTYPE html>: le indica al navegador que el documento usa HTML5.
  • <html>: elemento raíz que envuelve todo el documento.
  • <head>: contiene metadatos, el título de la pestaña, y enlaces a CSS o fuentes; no se muestra directamente en la página.
  • <body>: contiene todo el contenido visible de la página.

Atributos

Las etiquetas pueden llevar atributos, que agregan información adicional. Se escriben dentro de la etiqueta de apertura, como pares nombre="valor". 

<a href="https://ejemplo.com" target="_blank">
    Visitar sitio
</a>

Etiquetas HTML más Utilizadas

Texto 

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<p>Un párrafo de texto.</p>
<strong>Texto en negrita</strong>
<em>Texto en cursiva</em>
<br> <!-- salto de línea -->

HTML define seis niveles de encabezado, de <h1> (el más importante) hasta <h6> (el menos importante). Se recomienda usar un solo <h1> por página y mantener el orden jerárquico sin saltarse niveles.

Listas 

<ul>
    <li>Elemento sin orden</li>
    <li>Otro elemento</li>
</ul>

<ol>
    <li>Primer paso</li>
    <li>Segundo paso</li>
</ol>

Enlaces e imágenes 

<a href="pagina.html">Ir a otra página</a>

<img src="foto.jpg" alt="Descripción de la imagen">

El atributo alt es obligatorio por buenas prácticas: describe la imagen para personas que usan lectores de pantalla y se muestra si la imagen no llega a cargar.

Contenedores genéricos 

<div>
    Bloque genérico (ocupa toda la línea)
</div>

<span>
    Texto en línea (no rompe el flujo)
</span>

Etiquetas Semánticas

Además de <div> y <span>, HTML5 incluye etiquetas que describen el significado de cada sección de la página, no solo su apariencia. Esto mejora la accesibilidad y el posicionamiento en buscadores (SEO).

Etiqueta Uso
<header> Encabezado de la página o de una sección
<nav> Bloque de enlaces de navegación
<main> Contenido principal de la página (uno solo por documento)
<section> Agrupa contenido relacionado dentro de un tema
<article> Contenido independiente y reutilizable (una publicación, una noticia)
<aside> Contenido secundario, como una barra lateral
<footer> Pie de página o de una sección 
<header>
    <h1>Mi Blog</h1>
</header>

<nav>
    <a href="#inicio">Inicio</a>
</nav>

<main>
    <article>
        <h2>Título del artículo</h2>
        <p>Contenido del artículo...</p>
    </article>
</main>

<footer>
    <p>© 2026</p>
</footer>

Formularios en HTML

Los formularios permiten que el usuario envíe información: un texto, una selección, un archivo. Se construyen con la etiqueta <form> y distintos tipos de <input>. 

<form>

    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">

    <label for="correo">Correo:</label>
    <input type="email" id="correo" name="correo">

    <label for="mensaje">Mensaje:</label>
    <textarea id="mensaje" name="mensaje"></textarea>

    <label for="tema">Tema:</label>
    <select id="tema" name="tema">
        <option value="python">Python</option>
        <option value="ciberseguridad">Ciberseguridad</option>
    </select>

    <input type="checkbox" id="acepto" name="acepto">
    <label for="acepto">Acepto los términos</label>

    <button type="submit">Enviar</button>

</form>

Tipos de input más comunes

  • text: texto de una línea.
  • email: valida que el formato sea un correo.
  • password: oculta el texto escrito.
  • number: solo acepta números.
  • checkbox: casilla de verificación.
  • radio: selección única entre varias opciones.
  • date: selector de fecha.
  • submit: botón que envía el formulario.

La etiqueta <label> es importante por accesibilidad: conecta un texto descriptivo con su campo mediante el atributo for, que debe coincidir con el id del input.

Tablas en HTML

Las tablas se utilizan para mostrar datos organizados en filas y columnas. 

<table>

    <tr>
        <th>Lenguaje</th>
        <th>Tipo</th>
    </tr>

    <tr>
        <td>Python</td>
        <td>Interpretado</td>
    </tr>

    <tr>
        <td>C++</td>
        <td>Compilado</td>
    </tr>

</table>
  • <table>: contenedor de toda la tabla.
  • <tr>: define una fila (table row).
  • <th>: celda de encabezado (table header).
  • <td>: celda de datos (table data).

CSS: Sintaxis y Formas de Aplicarlo

CSS (Cascading Style Sheets) es el lenguaje que define la apariencia visual de una página HTML: colores, tamaños, espaciados, tipografía y disposición de los elementos.

Sintaxis básica 

selector {
    propiedad: valor;
    propiedad: valor;
}

p {
    color: #333;
    font-size: 16px;
}

Formas de aplicar CSS

1. CSS en línea (inline) 

<p style="color: red;">Texto rojo</p>

2. CSS interno 

<head>
    <style>
        p {
            color: red;
        }
    </style>
</head>

3. CSS externo (recomendado) 

<head>
    <link rel="stylesheet" href="style.css">
</head>

Un archivo CSS externo es la práctica recomendada, porque separa el contenido (HTML) del diseño (CSS) y permite reutilizar los mismos estilos en varias páginas, como se hace en este mismo blog con el archivo style.css.

Selectores CSS

Selector Ejemplo Selecciona
Etiqueta p { } Todos los <p>
Clase .card { } Todo elemento con class="card"
ID #inicio { } El elemento con id="inicio"
Descendiente nav a { } Los <a> dentro de <nav>
Universal * { } Todos los elementos

Una clase (.clase) puede repetirse en muchos elementos de la página, mientras que un ID (#id) debe ser único dentro de todo el documento.

El Modelo de Caja (Box Model)

En CSS, todo elemento se representa como una caja rectangular formada por cuatro capas, de adentro hacia afuera:

  • content: el contenido en sí (texto, imagen).
  • padding: espacio interno entre el contenido y el borde.
  • border: el borde de la caja.
  • margin: espacio externo entre esta caja y las demás.
.caja {
    width: 200px;
    padding: 20px;
    border: 2px solid #1f6feb;
    margin: 10px;
}

Por defecto, el ancho y alto que se definen solo aplican al contenido, y el padding y el borde se suman aparte. Una práctica muy común es usar: 

* {
    box-sizing: border-box;
}

Con esto, el padding y el borde quedan incluidos dentro del ancho definido, lo que hace mucho más predecible el tamaño final de cada caja.

Flexbox

Flexbox es un sistema de CSS diseñado para organizar elementos en una fila o columna, distribuyendo el espacio disponible entre ellos de forma flexible. 

.contenedor {
    display: flex;
    justify-content: center;
    align-items: center;
    gap: 10px;
}

Propiedades principales del contenedor

  • display: flex; activa flexbox.
  • flex-direction: row (fila) o column (columna).
  • justify-content: alinea los elementos en el eje principal (por ejemplo, horizontal).
  • align-items: alinea los elementos en el eje transversal (por ejemplo, vertical).
  • gap: espacio entre elementos.

Este mismo blog utiliza flexbox para centrar los enlaces de la barra de navegación y para acomodar las etiquetas de categorías en el encabezado.

CSS Grid

CSS Grid permite organizar elementos en una cuadrícula de filas y columnas, ideal para estructuras de página completas. 

.contenedor {
    display: grid;
    grid-template-columns: 1fr 1fr 1fr;
    gap: 20px;
}

En este ejemplo, 1fr 1fr 1fr crea tres columnas de igual ancho ("fr" significa fracción del espacio disponible). Grid es especialmente útil cuando se necesita controlar filas y columnas al mismo tiempo, mientras que Flexbox suele ser más sencillo para organizar elementos en una sola dirección.

Pseudoclases y Pseudoelementos

Pseudoclases

Aplican un estilo según el estado de un elemento. 

a:hover {
    color: #1557b0;
}

button:focus {
    outline: 2px solid blue;
}

li:first-child {
    font-weight: bold;
}

Pseudoelementos

Permiten insertar contenido o dar estilo a una parte específica de un elemento, sin necesidad de agregar una etiqueta HTML adicional. 

p::first-letter {
    font-size: 2em;
}

.tag::before {
    content: "# ";
}

Responsive Design

El diseño responsivo busca que una página se vea y funcione bien en cualquier tamaño de pantalla, desde un celular hasta un monitor grande.

Media Queries

Permiten aplicar reglas de CSS distintas según el ancho de la pantalla. 

@media (max-width: 768px) {
    nav ul {
        flex-direction: column;
    }
}

En este ejemplo, cuando la pantalla mide 768px o menos (como un celular o tablet), los enlaces del menú pasan de mostrarse en fila a mostrarse en columna.

Unidades relativas

Para un diseño responsivo también ayuda usar unidades que se adaptan, en vez de valores fijos en píxeles:

  • %: relativo al elemento contenedor.
  • vw / vh: relativo al ancho / alto de la ventana del navegador.
  • rem: relativo al tamaño de fuente base del documento.

JavaScript: Fundamentos

Mientras que HTML define el contenido y CSS el diseño, JavaScript es el lenguaje que le da interactividad a una página: permite reaccionar a clics, validar formularios, mostrar u ocultar contenido y modificar la página después de que ya fue cargada.

Cómo incluirlo 

<script src="script.js"></script>

Variables 

let nombre = "Joseph";
const edad = 20;

console.log(nombre, edad);

Funciones 

function saludar(nombre) {
    return "Hola, " + nombre;
}

console.log(saludar("Mundo"));

Eventos

Un evento es una acción que ocurre en la página, como un clic o el envío de un formulario. JavaScript permite "escuchar" esos eventos y reaccionar a ellos. 

const boton = document.querySelector("button");

boton.addEventListener("click", function () {
    alert("Botón presionado");
});

El DOM

El DOM (Document Object Model) es la representación en memoria de la página HTML que el navegador construye a partir del documento. JavaScript utiliza el DOM para leer y modificar el contenido de la página de forma dinámica, sin necesidad de recargarla.

Seleccionar elementos 

const titulo = document.querySelector("h1");
const parrafos = document.querySelectorAll("p");

Modificar contenido 

titulo.textContent = "Nuevo título";
titulo.style.color = "blue";

Crear y agregar elementos 

const nuevoParrafo = document.createElement("p");
nuevoParrafo.textContent = "Este párrafo fue creado con JavaScript";

document.body.appendChild(nuevoParrafo);

Gracias al DOM, una página puede actualizarse en respuesta a las acciones del usuario: agregar tareas a una lista, mostrar mensajes de validación en un formulario, o cambiar de sección sin recargar toda la página.

Dato Interesante

JavaScript fue creado por Brendan Eich en tan solo 10 días, en 1995, mientras trabajaba en Netscape. A pesar de su nombre, no tiene una relación directa con el lenguaje Java; el nombre fue una decisión de mercadeo de la época, cuando Java era muy popular.
